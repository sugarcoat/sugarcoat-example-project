Making UI documentation a bit sweeter ✨

Sugarcoat was created to enable developers to produce rich UI documentation easily and with minimal up-keep. Sugarcoat works by parsing project files for documentation comments (similar to JavaDoc, JSDoc, etc.) and generates HTML or JSON that is organized and easy to read. Sugarcoat allows developers and designers to access up-to-date previews of UI elements, page components, project specific colors and typography, all in one place.

Features

Universal Comment Styles

Use the same JSDoc commenting syntax across all file types. In HTML files, just wrap your comments in a standard HTML comment block. If you've got inline comments, Sugarcoat will ignore them.

Easy-to-identify component states

You can tell sugarcoat that there are modifier states in your css, right within your comment block! Sugarcoat will help highlight and display them for extra readability.

You decide where your library lives

Sugarcoat will never impose a folder structure on you.

Prefixed Variables? No Problem.

Sugarcoat will still understand your variables if they're SCSS, LESS, or even future specs: --my-var

No Style Bleed

And iFrames means no re-calculating dimensions for component example containers.

No weird comment delimiters

Sugarcoat will parse all comment blocks in a specified file. But hey, if you like weird delimiters, you can customize those too!
